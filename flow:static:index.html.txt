1. INTERACTION MODEL (SOURCE OF TRUTH)
────────────────────────────────

Flow v1 is PUSH-TO-TALK ONLY:

• User presses and holds the orb → app enters Listening.
• User releases → app:
  - stops listening,
  - transcribes,
  - translates,
  - then speaks.
• While the app is in Speaking, a new press & hold = INTERRUPT:
  - Stop TTS playback immediately on the client.
  - If the protocol expects a playback-done message, send it.
  - Transition to Listening and start a new turn.

There is NO AUTO / always-on mode in the v1 UI.
There must be NO VAD sliders or “reliability” knobs in the visible UI.

Minimum recording duration:
• If the user holds for less than 500ms:
  - Discard the turn (no server call).
  - Return to Idle.
  - Show a brief hint near the orb: “Hold a bit longer to speak.”
