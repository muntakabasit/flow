# FLOW_SUDANESE_ARABIC_STT_FEASIBILITY_GATE_V1 — Test Kit

**Purpose:** Determine whether Flow's Whisper layer (`mlx-community/whisper-medium`)
can reliably hear one available Sudanese-Arabic speaker. **STT only.** This kit
does not touch `server_local.py`, translation, language locks, TTS, or EN↔PT.

**Known constraint (documented, not changed):** Flow's live pipeline gates on
`ALLOWED_LANGS = ["en","pt"]` — Arabic detections currently route to Lane C
(repeat-request) by design. This gate therefore tests the raw Whisper layer
directly, offline, isolated from the EN/PT rescue chain.

---

## 1. Recording protocol (done by/with the speaker)

Record **15 samples** on any phone, quiet room, natural speech, ~3–15 s each
(longer conversational samples may run to ~30 s). Any format (m4a/wav/mp3).

| IDs | Count | Scenario |
|-----|-------|----------|
| S01–S03 | 3 | Ordinary Sudanese-Arabic statements |
| S04–S06 | 3 | Questions |
| S07–S09 | 3 | Longer natural conversational speech |
| S10–S11 | 2 | Hesitation / self-correction mid-utterance |
| S12–S13 | 2 | Arabic–English code-switched utterances |
| S14–S15 | 2 | Names, places, culturally specific terms |
| M01–M02 | 2 | **Control:** Modern Standard Arabic (read a news sentence) |

Place files in `tests/arabic_stt_gate/audio/` named `S01.m4a` … `M02.m4a`.

## 2. Fill the manifest

Edit `manifest.csv`: one row per file with the speaker's **human reference
transcription** (what was actually said, in Arabic script, code-switch words
kept in Latin script) BEFORE running the probe. This ordering guarantees the
reference is not influenced by Whisper's output.

## 3. Run the probe (on the Mac Mini — same model/runtime as Flow)

```bash
cd /Users/kulturestudios/ofa_jack_agent/projects/flow
python3 tests/arabic_stt_gate/run_probe.py
```

Each sample is transcribed twice: auto-detect and forced `ar`. Raw output is
**appended** to `raw_whisper_output.jsonl` — the script refuses to overwrite
existing runs (custody law).

## 4. Judge

Open `evidence.csv` (generated by the probe, one row per sample with raw
outputs embedded). With the speaker, fill the judgment columns:
`meaning_preserved` (yes/partial/no), `dialect_words_preserved`,
`code_switch_preserved`, `names_places_preserved`, `uncertainty_notes`,
`speaker_judgment`.

## 5. Custody chain

```
audio/Sxx.m4a  →  raw_whisper_output.jsonl (append-only, never edited)
               →  manifest.csv (human reference, written before probe)
               →  evidence.csv (comparison judgments)
```

## Pass condition

- ≥ 12/15 primary samples preserve usable meaning
- no systematic dialect-collapse pattern (e.g. everything normalized to MSA/Egyptian)
- code-switching visibly evaluated (S12–S13)
- speaker confirms broad faithfulness
- every failure traceable to its source audio file
