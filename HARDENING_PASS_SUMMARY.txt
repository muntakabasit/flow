WHAT CHANGED
─────────────────────────────────────────────────────────────────────────────

CHANGE 1: Runtime Mode Switching
  Before: RELIABILITY_MODE = "stable"  # global, hardcoded
  After:  Per-session mode from query (?mode=stable|fast) or localStorage

  Implementation:
  • Server: MODE_CONFIG dict with per-mode parameters
  • Server: Accepts mode_preference message from client
  • Client: Query param parser → localStorage → default fallback
  • Client: Sends mode preference on session init
  • Response: Server echoes mode_confirmed with applied timeout

CHANGE 2: Language Normalization
  Before: detected_lang passed raw (pt, pt-br, ru, it, sq, etc.)
  After:  normalize_lang() → {en, pt-BR} or None (unsupported ignored)

  Implementation:
  • Function: normalize_lang(raw_lang) in server
  • All variants: pt, pt-pt, pt-br → pt-BR
  • Unsupported: ru, it, sq, es, fr → None (skipped)
  • Output state: ONLY en or pt-BR (never raw)
  • Translation target: Always deterministic opposite

CHANGE 3: WebSocket Telemetry + Resilience
  Before: One-off close → immediately OFFLINE/FAILED
  After:  Grace policy: 2 consecutive failures before FAILED

  Implementation:
  • Captured telemetry: code, reason, visibility, navigator.onLine, count
  • failedReconnectCount tracker on client
  • Graceful OFFLINE state on first drop
  • FAILED only after 2 consecutive exhausted backoffs
  • Full diagnostic logging for debugging

CHANGE 4: Session Stability
  Before: Global SILENCE_DURATION_MS, timers might leak
  After:  Per-session MODE_CONFIG values, explicit cleanup

  Implementation:
  • Session-specific silence, keepalive interval, keepalive timeout
  • Cleanup on onclose: clearTimeout + clearInterval
  • sessionWanted preserved across reconnects
  • failedReconnectCount reset on successful connect

├─ en, en-US, en-GB → "en"
├─ pt, pt-BR, pt-PT, pt-pt → "pt-BR"

Reconnect Grace Policy
├─ Code 1000: IDLE (clean close)
├─ Code 1006+: OFFLINE (attempt recovery)
│   └─ If MAX_BACKOFF_ATTEMPTS exhausted: Check failedReconnectCount
│       ├─ failedReconnectCount < 2: Stay OFFLINE, try again
│       └─ failedReconnectCount >= 2: FAILED (give up)
└─ On successful flow.ready: Reset failedReconnectCount

KNOWN RISKS
─────────────────────────────────────────────────────────────────────────────

Risk 1: Whisper Misdetection Variance (MINOR)
  Chance: ~5% with clear speech
  Impact: Might switch language if 3+ consecutive misdetections
  Mitigation: normalize_lang() ignores unsupported; filters clearly
  Status: ✅ Acceptable

Risk 2: Mode Switch Packet Loss (VERY LOW)
  Chance: ~1% under normal network
  Impact: Client sends mode_preference but server doesn't receive
  Mitigation: Server falls back to DEFAULT_RELIABILITY_MODE
  Status: ✅ Acceptable (user can reconnect with ?mode=X)

Risk 3: localStorage Unavailable (LOW)
  Chance: iOS private browsing, some restricted contexts
  Impact: Mode preference not persisted
  Mitigation: Falls back to query param or default
  Status: ✅ Acceptable (app mode usage typically persists)

Risk 4: Multiple Tabs Conflict (LOW)
  Chance: Simultaneous connections from multiple tabs
  Impact: Last-write-wins; tabs could fight over mode
  Mitigation: Each tab can set its own; sequential sessions work fine
  Status: ✅ Acceptable (rare in single-user scenario)
